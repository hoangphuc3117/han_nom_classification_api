# Han Nom Classification API - Docker Auto-Build Setup

🚀 **Complete Docker automation for Han Nom Classification API with PaddleOCR**

This guide provides everything you need to build, deploy, and manage your Han Nom Classification API using Docker with full automation.

## 🛠️ Prerequisites

- Docker Desktop or Docker Engine (20.10+)
- Docker Compose (v2.0+)
- At least 4GB of available RAM
- Model files in the `./models` directory

## 🚀 Quick Start Options

### Option 1: Use the Quick Setup Script (Recommended)

Make the script executable:
```bash
chmod +x quick-docker.sh
```

**Development Mode** (with hot reload):
```bash
./quick-docker.sh dev
```
✅ API available at: http://localhost:8001/docs

**Production Mode**:
```bash
./quick-docker.sh prod
```
✅ API available at: http://localhost:8000/docs

**Production with Nginx**:
```bash
./quick-docker.sh nginx
```
✅ API available at: http://localhost/docs

### Option 2: Use Makefile Commands

```bash
# Development setup
make dev-setup

# Production setup  
make prod-setup

# With docker-compose
make docker-compose-up
```

### Option 3: Direct Docker Compose

```bash
# Development environment
docker compose --profile dev up -d --build

# Production environment
docker compose up -d --build

# Production with nginx reverse proxy
docker compose --profile production up -d --build
```

## 🔧 Configuration

### Environment Variables
- `PYTHONPATH=/app`
- `PYTHONDONTWRITEBYTECODE=1`
- `PYTHONUNBUFFERED=1`

### Model Configuration
Models được mount từ thư mục `./models` vào container tại `/app/models`

## 🐛 Troubleshooting

### Container không start
```bash
# Kiểm tra logs
docker-compose logs -f

# Kiểm tra container status
docker-compose ps
```

### Port conflict
```bash
# Thay đổi port trong docker-compose.yml
ports:
  - "8001:8000"  # Thay vì 8000:8000
```

### Model files missing
```bash
# Đảm bảo models folder tồn tại
ls -la models/

# Kiểm tra mount trong container
docker-compose exec han-nom-api ls -la /app/models/
```
